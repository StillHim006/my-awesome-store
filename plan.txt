This is a smart move. Using Google Sheets as your backend is a "pro-level" way to handle an e-commerce landing page for free. It gives you a structured database where you can see all your orders, phone numbers, and timestamps in one place.

## Phase 1: The "Brain" (Google Sheets & Apps Script)

Instead of PHP, we use a Google Script to receive data.

1. **The Spreadsheet:** Create a Google Sheet named "Store Orders." In the first row, create these headers: `Timestamp`, `Name`, `Phone`, `Location`, `Product`, `Price`, `Quantity`.
2. **The Script:** Go to **Extensions > Apps Script**. Paste the code I provided earlier (the `doPost` function).
3. **The Deployment:**
* Click **Deploy > New Deployment**.
* Select **Web App**.
* Set "Execute as" to **Me**.
* Set "Who has access" to **Anyone** (this allows your website to send data without a login).
* **Copy the Web App URL** (e.g., `https://script.google.com/macros/s/.../exec`).



---

## Phase 2: The "Storefront" (GitHub Files)

You only need two HTML files to keep things simple and fast.

### 1. `index.html` (The Landing Page)

Show all your products here. Every "Buy" button should link to your template page with an ID:

```html
<a href="product.html?id=watch_01">Buy Luxury Watch</a>
<a href="product.html?id=shoes_02">Buy Sport Shoes</a>

```

### 2. `product.html` (The Dynamic Template)

This page stays the same, but the content changes based on the URL.

* **The Data:** Create a `products.json` file where you store descriptions, prices, and image paths for every ID.
* **The JavaScript:** Use `URLSearchParams` to detect which product was clicked, fetch the data from your JSON, and update the `<img>` and `<h1>` tags automatically.
* **The Logic:** Calculate the price automatically ($Price \times Quantity$) as the user changes the number of items.

### 3. The Order Form

Your form in `product.html` must have `name` attributes that match your Google Sheet headers **exactly** (case sensitive).

```html
<form action="YOUR_WEB_APP_URL" method="POST">
  <input type="text" name="Name" required>
  <input type="text" name="Phone" required>
  <input type="hidden" name="Product" id="sheet_product">
  <input type="hidden" name="Price" id="sheet_price">
  <button type="submit">Confirm Order</button>
</form>

```

---

## Phase 3: The "Address" (Free Domain)

Since you want a free domain:

1. **Default:** Your site lives at `yourname.github.io/project-name`.
2. **Pro Option:** Since you are using GitHub, apply for the **GitHub Student Developer Pack** (if eligible) to get a free **.me** or **.tech** domain for a year.
3. **The Subdomain Trick:** If not a student, search for **is-a.dev**. It’s a community-run project where you can get a free `yourname.is-a.dev` domain by making a small request on their GitHub.

---

## Phase 4: Final Launch Checklist

1. **Push to GitHub:** Upload `index.html`, `product.html`, `script.js`, and `products.json`.
2. **Enable Pages:** In your Repo Settings > Pages, select the `main` branch.
3. **Test the Loop:** Open your site, click a product, fill the form, and check if the row appears in Google Sheets.
